## Что сделано в текущей сессии

### Health / Combat (цепочка последовательная)

- **`Health/Health.cs`** — HP + i-frames через `Timer`. i-frames = 0.5с.
  - События: `Damaged(int)`, `Died()`, `ValueChanged(int)`, `InvulnerabilityEnded()`
  - `Start()` стреляет `ValueChanged` однократно (для UI)
  - UI-независим (нет TMP_Text внутри)
- **`Health/Healer.cs`** — реген ПОСЛЕ окончания i-frames (не параллельно).
  - Подписан на `Health.InvulnerabilityEnded`, не на `Damaged`
  - Цепочка: урон → i-frames (0.5с) → `InvulnerabilityEnded` → regen-таймер (5с) → `Heal(до MaxValue)`
- **`Combat/Attacker.cs`** (неймспейс `NPC.Enemy`) — автономная атака врага.
  - Работает в `Update`, никто не дёргает
  - Использует `TargetSensor` + `ITarget.Health.TakeDamage(1)`
  - Cooldown через `Timer` (0.5с)
  - Урон = 1 (константа)

### ITarget / Sensors

- **`Interfaces/ITarget.cs`** — интерфейс `{ Transform, Health }`
- **`Detectors/TargetSensor.cs`** (бывший `PlayerSensor`, переименован) — OverlapSphere ищет `ITarget` на коллайдерах
  - События: `TargetEntered(ITarget)`, `TargetExited()`
  - Свойства: `IsTargetInRange`, `DetectedTarget`
- **`Player/Player.cs`** — реализует `ITarget`
- **Enemy/Prey** — используют `TargetSensor` (старый `PlayerSensor` удалён)

### PlayerScaler

- Логарифм заменён на **линейное деление**: `multiplier = 1 + (current - startMass) / _scaleDivisor`
- Делитель настраивается в инспекторе (default `20f` → 5% за итем при mass=1)
- `_growthRate` поле удалено

### Rotator

- В `Rotate(Vector3 direction)` обнуляет `direction.y` перед `LookRotation` — поворот только вокруг Y
- Параметр копируется по значению, `Mover` получает полный 3D-вектор как раньше

### NPC Behaviors

- `Wander`, `Chaser`, `Fleer` теперь зовут `_rotator.Rotate(direction)` после `_mover.Move(direction)`
- У `Fleer` поворот по `-direction` (в сторону движения = от игрока)

### UI слой

- Новая папка `Assets/Scripts/UI/`, неймспейс `UI`
- **`UI/HealthUI.cs`** — один `TMP_Text`, формат `"current / max"`. Подписан на `Health.ValueChanged` + `Died`
- **`UI/TimerUI.cs`** — один `TMP_Text`, формат `0.0` (1 знак). Подписан на `Timer.Ticked`
- **`Game/Timer.cs`** — выкинут весь UI. Добавлено событие `Ticked(float remaining)` + свойство через событие
- `Health` уже не имел UI-ссылок (проверено)

### Звук (цепочка реактивная, без singleton)

- **`Saves/SavesYG_Audio.cs`** — partial `SavesYG` + `musicVolume`, `sfxVolume` (default 0.8). Хранение через YG2.saves, не PlayerPrefs.
- **`Audio/AudioMixerController.cs`** — мост YG2.saves ↔ AudioMixer. Группы: `Music`, `SFX`. Exposed params: `MusicVolume`, `SFXVolume`. Подписан на `YG2.onGetSDKData`, при инициализации читает сейв и применяет. Методы `SetMusicVolume/SFXVolume(float 0..1)` — clamp, запись в `YG2.saves`, `YG2.SaveProgress()`. Геттеры `MusicVolume`/`SFXVolume` для UI слайдеров.
- **`Audio/PlayerPickupSound.cs`** — `[RequireComponent(AudioSource)]`, подписан на `Collector.ItemCollected` → PlayOneShot.
- **`Audio/PlayerDamageSound.cs`** — подписан на `Health.Damaged`.
- **`Audio/PlayerDeathSound.cs`** — подписан на `Health.Died`.
- **`Audio/EnemyAttackSound.cs`** — подписан на `Attacker.AttackPerformed` (новое событие).
- **`Audio/FlyingCubeSound.cs`** — `[RequireComponent(FlyingCube)]`, подписан на `FlyingCube.Arrived`.
- **`Audio/TimerTickSound.cs`** — подписан на `Timer.Ticked` + `Finished`. Запускает луп при `remaining <= 20` (порог настраивается), стоп при Finished или выходе из диапазона.
- **`Audio/LevelMusicPlayer.cs`** — Start() → loop Play(). На отдельном GameObject `[AudioMusic]`.
- **`Audio/UIButtonSound.cs`** — `[RequireComponent(Button)]`, подписан на `Button.onClick` через AddListener/RemoveListener (UnityEvent — API Unity, не наш).
- **`Audio/AudioSettingsPanel.cs`** — два `Slider` (0..1) → `AudioMixerController.Set*Volume`.
- **`NPC/Enemy/Attacker.cs`** — добавлено `public event Action AttackPerformed` + инвокация после `TryApplyDamage`. Остальная логика не тронута.

### Архитектурные решения по звуку

- Без singleton и static. Каждый звук = отдельный компонент.
- Без базового класса. У каждого звука своё событие, свой lifecycle.
- Источник истины — `YG2.saves`. Микшер получает значения через `AudioMixerController`.
- Звуковые компоненты не зависят от контроллера — только через `AudioMixerGroup` в SerializeField.
- Каждый звуковой компонент имеет свой `AudioSource` через `[RequireComponent]`. У Player будет 3 AudioSource — это норм для Unity.

### Что осталось руками в Editor

- `Assets/Audio/MasterMixer.mixer` — создать, группы `Music`/`SFX`, expose params `MusicVolume`/`SFXVolume`.
- AudioClips — закинуть в `Assets/Audio/Clips/`, прокинуть в компоненты.
- В сцене: GameObjects `[AudioCore]`, `[AudioMusic]`, расставить компоненты на Player/Enemy/FlyingCube/Timer/Button.

# GDD: Mad Slime

## 1. Анализ рефа

### 1.1. Краткое описание реф-игры
Аркадная мобильная игра в мультяшном low-poly стиле.
Игрок управляет слаймом, который поглощает объекты в современных городских и интерьерных локациях. Основной прогресс строится на выполнении квоты уровня через поглощение определённых предметов.
Сеттинг: современный город, квартиры, улицы, промышленные зоны.
Визуальный стиль: low-poly, мультяшный.

### 1.2. Основная механика гейплея и управление
Основная механика: втягивание объектов слаймом и выполнение квоты уровня.
Игрок управляет движением слайма.
Игровой цикл:
- перемещение по уровню
- втягивание объектов в радиус
- поглощение предметов
- поиск нужных объектов для квоты
- накопление прогресса
- переход на следующий уровень
В конце уровня все поглощённые объекты конвертируются в кубы/очки:
- предметы из квоты дают полную награду
- остальные предметы дают уменьшенную награду
Если игрок набирает больше необходимой квоты, излишек переводится в игровую валюту.
Управление:
- телефон — виртуальный джойстик и кнопки
- ПК — клавиатура

### 1.3. Дополнительные механики
- система квот на уровень
- система времени на прохождение уровня
- рост слайма внутри уровня для визуального отображения прогресса
- уровни, привязанные к размеру сущности
- разные типы объектов по размеру и ценности
- движущиеся объекты (люди, животные, техника)
- статичные объекты окружения
- враги и опасные объекты
- дебаффы от врагов: увеличение требуемой квоты, потеря прогресса
- смерть при попадании в опасные зоны или получении большого количества урона
- система валюты
- система навыков
- система ребитха
- переход между уровнями с изменением окружения и масштаба

### 1.4. Притягательные аспекты
- простая и понятная core-механика
- визуально приятное втягивание объектов
- постоянный прогресс внутри уровня
- чувство усиления через рост слайма
- поиск нужных объектов для выполнения квоты
- напряжение из-за таймера и врагов
- разнообразие окружения и размеров объектов
- возможность билдов через навыки

### 1.5. Промахи и риски
- риск перегрузки механиками
- возможный дисбаланс квоты и таймера
- слабая читаемость объектов при большом количестве предметов
- риск однообразия без разнообразных уровней
- сложность балансировки экономики
- сложность балансировки ребитха

## 2. ГДД планируемой игры

### 2.1. Описание игры
Игрок управляет слаймом неизвестного происхождения, который появляется в квартире.
Основная задача игрока — поглощать объекты, выполнять квоту уровня и переходить на новые локации.
Каждый уровень имеет:
- определённую квоту
- ограничение по времени
- набор объектов
- врагов и опасности
Игрок может поглощать любые объекты, но предметы, относящиеся квоте уровня, дают больше прогресса и награды.
Пример: на первом уровне квотой могут быть канцелярские предметы (ручки, карандаши, ластики).
Обычные предметы дают уменьшенную награду и помогают поддерживать темп.
Квотные предметы дают основной прогресс и позволяют завершить уровень.
После завершения времени:
- если квота выполнена → уровень пройден, лишний прогресс → валюта
- если квота не выполнена → поражение
Сеттинг: современный мультяшный low-poly город.

### 2.2. Структура игры

**Старт игры:** короткий комикс о появлении слайма в мире, затем первый уровень. Обучение отсутствует или минимальное.

**Основной игровой цикл:**
- игрок появляется на уровне
- получает квоту
- перемещается по карте
- поглощает объекты
- ищет нужные предметы
- избегает врагов и опасностей
- увеличивает прогресс квоты
- завершает уровень до окончания времени

**Уровни (пример прогрессии, ~10 штук):**
1. квартира
2. дом
3. улица
4. парковка
5. промзона
- каждый уровень соответствует размеру слайма
- с увеличением размера меняется масштаб объектов
- новый уровень = новое окружение и типы объектов
- сложность растёт

**Поражение:** не выполнил квоту / слишком много урона / смертельная зона → рестарт уровня.

**Метапрогрессия:**
- валюта за прохождение уровней
- навыки за валюту
- скины — дополнительно
- достижения — опционально

**Навыки (3 ветки):**
- **Выживание:** +max HP, сопротивление урону, шанс избежать урона
- **Мобильность:** +скорость, рывок
- **Поглощение:** +радиус притягивания, автопритягивание, тентакли

**Ребитх:** после прохождения игры активируется ребитх, который увеличивает сложность и добавляет модификаторы (меньше еды, больше врагов, опаснее уровни). Награда: новые модификаторы, косметика.

### 2.3. Визуальная составляющая
- Стиль: low-poly, мультяшный, упрощённые формы, яркие цвета, хорошая читаемость
- Слайм: визуально растёт, деформируется при движении, эффекты втягивания
- Окружение: квартиры, улицы, городские зоны, промышленные зоны
- Объекты: мелкие предметы, бытовые предметы, люди и животные, техника и транспорт, здания

### 2.4. Монетизация
- **Интерстишиал-реклама:** после завершения уровня, после поражения
- **Ревард-реклама:** возрождение, доп. валюта, временные бонусы, усиления внутри забега
  - замедление врагов, увеличение награды, усиление притягивания, ускорение движения
